# Services Hardening

Este proyecto documenta buenas prácticas para **hardening de servicios críticos** en sistemas Linux, incluyendo Apache, Nginx, MySQL/PostgreSQL y OpenSSH.  
Su objetivo es minimizar la superficie de ataque y garantizar configuración segura por defecto.

## Objetivos

- Proporcionar guía clara y defendible para hardening de servicios.
- Incluir ejemplos de permisos, firewall y logging.
- Scripts educativos para verificar configuración segura.

## Alcance

- Apache/Nginx
- MySQL/PostgreSQL
- OpenSSH

## Estructura del proyecto

- `apache-hardening.md` → Configuración segura de Apache.  
- `nginx-hardening.md` → Configuración segura de Nginx.  
- `mysql-hardening.md` → Configuración segura de bases de datos MySQL/PostgreSQL.  
- `scripts/` → Scripts de verificación de seguridad.
